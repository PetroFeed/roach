    queue is based on emitter-queue in order to run a job asynchronously.

    An event added is queued once a job has been added into the pending queue. It means you can subscribe to this event after it has been fired and the callback will still be executed.

        it("should queue events", function(done) {
      queue.add('event');
      queue.on('added event', function(val) {
        done();
      });
    });
